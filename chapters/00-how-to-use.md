# บท 0 — วิธีใช้หนังสือเล่มนี้

## อ่านยังไงให้ได้ผลจริง

pwn เป็นทักษะที่ **เข้าใจจากการเห็น memory จริง** ไม่ใช่จากการอ่านเฉยๆ
กฎเหล็ก 4 ข้อของเล่มนี้:

### 1. อย่าอ่านให้จบก่อนแล้วค่อยลงมือ
อ่านบทหนึ่ง → ลองทำโจทย์ที่เกี่ยวข้องทันที → ติดตรงไหนค่อยกลับมาอ่าน
สมองจำได้จากการ "เจอปัญหาแล้วแก้" มากกว่าการท่องล่วงหน้า

### 2. เปิด GDB ทุกครั้ง
ทุกบทที่พูดถึง memory ให้เปิด GDB ดูของจริงตาม
เห็น stack เลื่อน เห็น RIP เปลี่ยน จะเข้าใจเร็วกว่าอ่านคำอธิบาย 10 รอบ

### 3. เปิด writeup ได้ แต่พิมพ์เอง
ติดจนหมดหนทางแล้วเปิด writeup ได้ — แต่ห้ามก็อป
อ่านให้เข้าใจ ปิด แล้วพิมพ์ exploit ใหม่เองทั้งหมด

### 4. ทำซ้ำจนเป็นกล้ามเนื้อ
template pwntools, คำสั่ง GDB, ขั้นตอน [checksec](05-tools.md#1-checksec--ด่านแรกเสมอ) → cyclic → leak
พวกนี้ต้องทำจนไม่ต้องคิด เหมือนผูกเชือกรองเท้า

        ↓

        ↓
บท 9-10 (format string + GOT)  →  บท 11 (workflow) ทบทวน
        ↓
